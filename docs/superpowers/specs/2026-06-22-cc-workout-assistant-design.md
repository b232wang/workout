# CC 健身助手仓库 — 设计文档

**日期:** 2026-06-22
**状态:** 待用户确认

## 1. 背景与目标

把原本在 cowork 里做的「AI 健身助手」迁移到 Claude Code。动机:

- cowork 手机端访问不了;CC 远程会话(claude.ai/code、手机端)可以。
- 希望用更聪明的大脑(Opus)持续管理这个健身项目。

**目标:** 一个 git 仓库,作为个人健身的「知识库 + 记录本」。用户在手机或电脑上开 CC 会话,CC 读仓库文件即可扮演私人教练 + 训练记录员。

## 2. 用户背景(健身参数)

- 目标:增肌
- 水平:有一定基础
- 频率:每周 6 天,每次约 30 分钟
- 场地:Planet Fitness(器械 / 史密斯机 / 哑铃 / 绳索齐全,无自由杠铃深蹲架)+ 家
- 分化:推 / 拉 / 腿 三分化,每周跑两轮(每个肌群每周练 2 次)
- 选动作原则:优先在肌肉「拉长位」施加张力的动作(结合 2026 研究)

## 3. 核心架构:自解释仓库

**关键设计:** 仓库自带一份给 CC 的说明书(`CLAUDE.md`)。手机上新开的 CC 会话没有历史对话记忆,必须靠仓库文件自解释 —— `CLAUDE.md` 让任意新会话读完即进入「教练 + 记录员」角色。这是 cowork 给不了的能力。

**数据流:**

```
用户(手机 / CC 远程会话) ⇄ CC ⇄ 读写仓库内的 Markdown 文件
```

无后端、无部署、无数据库。所有状态都是 git 仓库里的 Markdown。

## 4. 文件结构

```
workout/
├── CLAUDE.md       # 给 CC 的说明书 + 角色 + 响应规约
├── 训练计划.md      # 推/拉/腿 三分化(末尾附营养与恢复要点)
├── 拉伸方案.md      # 练前动态热身 + 练后静态拉伸
├── 训练日志.md      # 训练记录(日期倒序,最新在顶部)
├── 动作库.md        # 每个动作的详细文字要领
└── docs/superpowers/specs/   # 本设计文档
```

### 各文件职责与格式

**`CLAUDE.md`** — 给 CC 的项目说明书,是整个设计的核心。包含:

- 项目是什么 + 用户背景(见第 2 节)
- CC 的角色:私人教练 + 记录员;语气简洁,中文
- **响应规约**(让无上下文的新会话也能正确工作):
  - 用户说「今天练 X / 练什么」→ 读 `训练计划.md` 对应日,输出动作 + 组数次数 + 一句要点;并从 `训练日志.md` 查同动作上次重量做提醒
  - 用户报数据(如「卧推 60 做了 10/10/8」)→ 解析后写入 `训练日志.md`:有当天条目则追加,没有则在文件顶部新建当天条目
  - 用户问「某动作怎么做」→ 读 `动作库.md` 给文字要领
  - 用户问「进步 / 某动作历史」→ 从 `训练日志.md` 提取该动作历史,给趋势
  - 用户说「换动作 / 不喜欢 X」→ 从 `训练计划.md` 该动作的备用里换,并更新计划文件
- 数据格式约定(日志格式、单位)

**`训练计划.md`** — 推 / 拉 / 腿 三天,每天若干动作。每个动作用卡片式(手机友好,不用宽表格):

```
### 上斜哑铃卧推
- 组数次数:3 × 8-12
- 目标肌:上胸
- 要点:[一句话]
- 备用:平板哑铃卧推 / 上斜史密斯推
```

末尾附一节简短「营养与恢复要点」(内容已有,对增肌重要,成本低,不单独成文件)。

**`拉伸方案.md`** — 练前动态热身、练后静态拉伸(含腿 + 上半身)。每个拉伸:部位、做法、时长。

**`训练日志.md`** — 训练记录,日期倒序(最新在顶部)。格式:

```
## 2026-06-22 · 推 · 第 N 次
- 上斜哑铃卧推 22.5kg × 12/10/8
- 坐姿绳索夹胸 15kg × 15/15/12
- 体感:[可选]
```

约定:哑铃记**单只**重量;重量单位 kg;组间次数用 `/` 分隔。

**`动作库.md`** — 所有出现过的动作(主选 + 备用,去重)的详细文字要领:发力、轨迹、常见错误、拉长位提示。按部位分组。纯文字,无图、无链接。

## 5. 内容来源

训练计划、拉伸、动作要领全部来自本次 cowork 对话记录(已包含完整的三分化计划、每个动作的两个备用、拉伸方案、动作要点、2026 研究依据、哈克深蹲 vs 杠铃深蹲等)。**迁移无损**,无需 cowork 那些 docx/pdf 文件。

## 6. 验收标准

成功 = 新开一个**无历史上下文**的 CC 会话,在 `workout` 目录里:

1. 说「今天练推」→ 正确输出推日计划 + 上次重量提醒(若日志有记录)
2. 报「卧推 60 做了 10/10/8」→ 正确写入 `训练日志.md`
3. 问「罗马尼亚硬拉怎么做」→ 从 `动作库.md` 给出文字要领
4. 问「卧推进步」→ 从日志提取历史并总结

## 7. 非目标(YAGNI)

- 不做 web app、不部署
- 动作不配图、不配视频链接(已确认:纯文字要领)
- 不做自动提醒 / 定时(如需要再单独加)
- 营养不单独成文件(放 `训练计划.md` 末尾一小节)

## 8. 实现步骤(交给 writing-plans 细化)

1. `git init`
2. 写 `CLAUDE.md`(角色 + 响应规约 + 格式约定)
3. 写 `训练计划.md`(从聊天记录搬,卡片式,附营养小节)
4. 写 `拉伸方案.md`
5. 写 `动作库.md`(去重后的全部动作要领)
6. 建 `训练日志.md`(含格式示例 / 表头)
7. 验收:模拟新会话,跑通第 6 节四个场景
